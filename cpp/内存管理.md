# 内存分区
## 栈
- 编译器管理分配和回收
- 存储局部变量、函数参数
## 堆
- 程序员管理，`new delete malloc free`进行分配和回收 
## 全局/静态存储区
- 初始化区：存储初始化的全局变量、静态变量
- 未初始化区：存储初始化的全局变量、静态变量
## 常量存储区
- 存储常量，不允许修改
## 代码区
- 存储程序的二进制代码

# new delete malloc free区别
 malloc与free是C++/C语言的标准库函数，new/delete是C++的运算符。
## new delete
- 运算符，编译器操作来执行构造和析构
- 按照数据类型申请和释放内存
## malloc free
- 库函数，编译器无法对其操作来进行构造和析构
- 按照字节长度申请和释放内存

# 内存泄漏
## 原因
程序员申请的空间，使用完后没有释放。
## 检测
- 观测swap
- /usr/bin/stat工具`netstat`、`vmstat`
- 内存调试，内存泄漏检查以及性能分析软件`valgrind`

# 内存对齐
- 偏移量必须是min(#pragma pack())的倍数
- 内存对齐可以加速计算
- 某些设备仅支持内存对齐的数据
